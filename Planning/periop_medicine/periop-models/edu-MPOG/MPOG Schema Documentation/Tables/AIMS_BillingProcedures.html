Columns
Name Data Type Max Length (Bytes) Allow Nulls Default Description
MPOG_Billing_Procedure_ID uniqueidentifier 16
(newsequentialid()) This is a GUID that is generated on insert. It is used to uniquely identifiy a row.
MPOG_Patient_ID uniqueidentifier 16
Identifies which patient in the MPOG database this procedure belongs to.
MPOG_Case_ID uniqueidentifier 16
Identifies which MPOG case this procedure belongs to. Note that this column is nullable due to difficulties in always matching procedures to cases.
AIMS_Encounter_Number varbinary(64) 64
Used to store the identifying "encounter" billing number. Given that this is PHI, this field must be encrypted or hashed.
MPOG_Billing_Source_CD tinyint 1
Identifies the source of the billing information (e.g. Anesthesia ProFee, Hospital Discharge, etc.). Refer to the MPOG_Enumerations table for a complete listing of possible values.
AIMS_Date_Of_Service_Start_DT datetime 8
The date of when the service began. What this date refers to can depend on what is being described (e.g when a patient was admitted, when anesthesia was started, etc.)
AIMS_Date_Of_Service_End_DT datetime 8
The date of when the service ended. This is applicable to things such as discharge date and anesthesia end.
AIMS_Procedure_Code varchar(16) 16
The value for the actual procedure code. The column is a varchar to handle various types of procedure codes.
MPOG_Lexicon_Type_CD tinyint 1
This code identifies which lexicon the AIMS_Diagnosis_Code is in (eg. ICD9, ICD10, CPT). Refer to the MPOG_Enumerations table for all possible values.
AIMS_Total_Anesthesia_Units smallint 2
The total number of anesthesia time units for the billing charge. Note that this only applies to time-based anesthesia procedure charges.
AIMS_Procedure_Priority tinyint 1
This column describes the importance of a given diagnosis (primary, secondary, ...).
