MPOG Server > MPOG_MAS > Tables

Name Description
dbo.AIMS_BillingDiagnoses Stores diagnosis codes that are pulled from the billing system.
dbo.AIMS_BillingModifiers Stores billing modifiers for procedures that are pulled from the billing system.
dbo.AIMS_BillingProcedures Stores procedure codes that are pulled from the billing system
dbo.AIMS_IntraopCaseInfo Each row in this table refers to a unique case. Almost all other AIMS_* tables refer to this table.
dbo.AIMS_IntraopInputOutputs Stores measurement data regarding interaoperative inputs and outputs.
dbo.AIMS_IntraopInputOutputTotals Stores the input/output totals for each case.
dbo.AIMS_IntraopMedications Stores all medications used during a case, including both boluses and infusions.
dbo.AIMS_IntraopNoteDetails Used to store individual components or additional properties of an intraoperative note. This is a child table of AIMS_IntraopNotes. This table may not be used depending on how the AIMS stores intraoperative notes.
dbo.AIMS_IntraopNotes Stores events and other notes recorded during the intraoperative period.
dbo.AIMS_IntraopPhysiologic Stores vital signs and other physiologic measurements recorded during an operation.
dbo.AIMS_IntraopStaff Stores when providers signed in and out of a case.
dbo.AIMS_LabValues Stores all labs for a patient one year prior to and following a case. Note that this table is linked directly to a patient instead of a case to prevent record duplication.
dbo.AIMS_Outcomes Stores the post-operative observations of a case.
dbo.AIMS_Patients This table stores all patients that have been imported into MPOG. Each patient should be a unique person i.e. the same patient can span multiple cases. Note that the PHI columns in this table are not transferred to the central MPOG database.
dbo.AIMS_PhiDictionary Stores the identifiers specific to the site's AIMS, such as the names of providers. Note that this table is not transferred to the central MPOG database.
dbo.AIMS_Preop Stores pre-operative observations for a case such as H&P.
dbo.AIMS_PreopDetails Used to store individual components or additional properties of a preop note. This is a child table of AIMS_Preop. This table may not be used depending on how the AIMS stores preoperative observations.
dbo.AIMS_Sites Stores information about the IV sites used during a case.
dbo.MPOG_Concept_Types All MPOG concepts have a type associated with them. This table stores those types and is a parent of MPOG_Concepts.

Example: The MPOG concept of "Blood Pressure" is a type of "Physiologic Observation."
dbo.MPOG_Concepts MPOG provides a large number of standardized "concepts" for various AIMS to map their concepts to. All of these concepts are stored in this table.
dbo.MPOG_Enumerations The various "MPOG Code" columns in the database (denoted by MPOG_..._CD) are documented in this table. While lacking a traditional foreign key constraint, a check constraint does enforce referential integrity to this table.
dbo.MPOG_Institutions This table stores the MPOG assigned identifiers for each institution. Note that each site's local copy of the MPOG database should contain only their assigned identifier.
dbo.MPOG_LabVariables Stores supplemental information about the standardized MPOG lab concepts.
dbo.MPOG_PhiDictionary MPOG provides a large number of common words, phrases, and medical terminology to support the PHI scrubbing application.
